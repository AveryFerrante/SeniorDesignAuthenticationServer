QR Code

QR code is now dynamic every load session and saved back end for up to 5 minutes before expiring. Each QR code is a unique 20 character string containing letters and numbers.
More information may be added to this QR code in the future, but for now, you should save this string, as it is required in future processing.




Creating an Account

Account creation now requires a username, password, and a passphrase. An account can be created by clicking on the QR Code from the homescreen or by visiting
http://54.213.49.145/signup.html




Logging in

To login, a POST request must be sent to login/ endpoint. The request must contain username and password and sent as the variables 'username' and 'password'
respectively. If the login is successful, a 'Granted' message is returned as well as a cookie that is to be saved on the device. Other possible responses are
'NotFound' and 'MissingArgument' respectively.




"Validating" a Session

Now, once the user scans the code on the phone, a POST request should be sent to qrScan/ endpoint. The request must send the cookie that has been saved from logging
in as well as the scanned code from the QR code. This code should be sent using the parameter name 'qrCode'. If everything validates correctly, you should receive
back the passphrase created when making the account. There is functionality to test this on the site right now. If you type in the scanned QR code into the correct box
and press login the server will alert with responses (ultimately you are wanting to see the passphrase you created with the account). For reasons, you may have to click
login a few times before the correct response will show in the dialog boxes. An example looks like this: Type in username, password, and scanned QR code and press login
button 2 - 3 times, reading the messagse that come from the server each time. This should allow you to "test" certain aspects. Downloading a 3rd party QR scanner can help
to display the QR code on your phone so you can type it into the computer easier.
